Batch Printing

Redesigning a warehouse print flow so operators complete batches faster with fewer errors.

Context

Warehouse teams print hundreds of labels per shift. The legacy batch screen stacked actions vertically, hid printer status, and forced operators to confirm the same data twice. Support tickets clustered around “wrong batch” and timeouts.

57%

Reduction in average batch completion time after launch

Problem

Operators lost context when switching between SKU lists and printer queues. Error recovery required leaving the flow. PM and ops wanted a single screen that matched how pickers actually work — scan, confirm, print, next.

Process

Shadowed three shifts, mapped the happy path and failure paths, and ran a clickable prototype with five operators. We prioritized: persistent batch summary, inline printer health, and one-tap reprint for the last job.

Solution

Split the UI into a fixed left rail (batch + progress) and a right workspace (line items + actions). Primary CTA stays anchored; destructive actions moved behind a confirm pattern shared with the design system.

Outcome

Average batch time dropped from 28s to 12s in the pilot site. Reprint-related tickets fell 34% in the first month. The patterns shipped back into the ops design system as “batch modules” for other tools.
